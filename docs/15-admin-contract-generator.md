# 15. 管理画面：業務委託契約書ジェネレータ

## このドキュメントの目的

`/admin/contracts` の契約書作成機能の概要と使い方、設計判断を記録します。

## 概要

撮影・Web 制作・運用案件の **業務委託契約書** を、フォーム穴埋め → 整形プレビュー → **印刷（PDF 保存）** で発行する管理者向けツール。見積書・請求書は弥生／Misoca で行うため、ここでは Misoca で作れない「契約書」だけを扱う。

| 項目 | 内容 |
|------|------|
| ルート | `/admin/contracts`（認証必須・`robots noindex`）|
| 出力方式 | **印刷 → PDF**（ブラウザの「PDF に保存」）。追加ライブラリ・サーバー生成なし |
| 保存 | **都度生成のみ**（DB なし）。フォーム状態はクライアントのみ |
| 構成 | `page.tsx`（サーバ）＋ `contract-generator.tsx`（クライアント） |

## 設計判断（なぜこの方式か）

- **印刷→PDF を採用**：撮影業の内部ツールで使用頻度は低く、`@react-pdf` 等のサーバー生成は日本語フォント同梱（数MB）と実装増になる。印刷経由なら依存ゼロ・日本語が完璧・Cloud Run に優しい。1クリック DL が必要になれば後から差し替え可能。
- **DB 非保存（MVP）**：契約は都度入力で足りるため Prisma モデルは追加しない。一覧・再利用が必要になれば `Contract` モデルを足す。

## 使い方

1. 管理画面 → **Contracts** を開く。
2. 左フォームに発注者名・委託料・業務内容・振込先などを入力（初期値は「候補者サイト＋撮影」想定でプリセット）。右側プレビューに即反映。
3. **運用・ホスティング条項**（第7条・契約期間条）、**政治・選挙特約** はチェックで ON/OFF。条番号は自動で振り直される。
4. 「**印刷 / PDF 保存**」→ 印刷ダイアログで「**PDF に保存**」を選択してダウンロード。

## 印刷時の挙動

- 管理ナビ（`admin/layout.tsx` の上部バー）と入力フォームは `print:hidden` で非表示。
- プレビュー（`.contract-paper`）のみ印刷。グリッドは `print:block` で1カラム化し、`@page { margin: 16mm }` を適用。

## 著作権条項について（文化庁マニュアル準拠）

第6条（著作権・利用範囲）は、文化庁「誰でもできる著作権契約マニュアル」（写真家向けアレンジ）を踏まえて設計：

- **利用許諾（留保）か譲渡か**を選択。留保時は**利用範囲を明示**（複製・公衆送信・展示・印刷物等のうち定めた範囲に限る）し、**独占／非独占**を指定（規定なき場合は非独占が原則）。
- **譲渡時は二次的著作物（著作権法第27条・第28条）を含む旨を明記**（明記しないと譲渡対象外と推定されるため）。
- **著作者人格権は譲渡不可**。譲渡選択時も「行使しない」とする。トリミング・色調等は軽微な調整のみ可、大幅な改変・合成は協議（同一性保持権への配慮）。
- **クレジット表記（氏名表示権）**の要否を選択。
- 追加条項：契約の解除（催告解除）／権利義務の譲渡制限（利用権の無断譲渡禁止・第63条3項）／契約内容の変更は書面。

## 契約プロセスで使う関連書類（この契約書以外）

| 書類 | 用途・備考 |
|------|-----------|
| 見積書 | 金額の正本。弥生／Misoca で発行（契約書本文は枠組み、金額は見積書に集約）|
| 発注書／注文請書 | 簡易な合意でも可。承諾書形式（一方署名）でも契約は成立 |
| 仕様・業務範囲確認書 | 撮影内容・サイト仕様の詳細を別紙化（スコープ膨張防止）|
| 写真利用許諾書 | 写真の単発利用だけのときの軽い許諾書形式 |
| 納品書・検収書 | 納品・検収の記録 |
| 請求書・領収書 | Misoca で発行 |
| **モデルリリース（肖像権同意）** | **人物撮影で重要。著作権とは別に被写体の肖像権・パブリシティ権の同意が必要な場合がある（マニュアルは著作権のみ）** |
| 撮影許可・会場使用許可 | イベント・施設での撮影時 |
| NDA（秘密保持契約） | 必要に応じて単独締結 |
| **保守運用契約書** | **公開後の運用代行は別契約。`/admin/maintenance` で発行（[17章](./17-admin-maintenance-contract.md)）** |

参考：文化庁「誰でもできる著作権契約マニュアル」／文化庁「著作権契約書作成支援システム」。

## 注意・今後

- 出力はあくまでひな型。重要・高額・継続契約は専門家確認を推奨（本ツールは法的助言ではない）。
- 入力値の保存（DB 化）、1クリック PDF DL（`@react-pdf`）、見積書連携、上記関連書類のジェネレータ化は将来の拡張候補。
