# 15. 管理画面：業務委託契約書ジェネレータ

## このドキュメントの目的

`/admin/contracts` の契約書作成機能の概要と使い方、設計判断を記録します。

## 概要

撮影・Web 制作・運用案件の **業務委託契約書** を、フォーム穴埋め → 整形プレビュー → **印刷（PDF 保存）** で発行する管理者向けツール。見積書・請求書は弥生／Misoca で行うため、ここでは Misoca で作れない「契約書」だけを扱う。

| 項目 | 内容 |
|------|------|
| ルート | `/admin/contracts`（認証必須・`robots noindex`）|
| 出力方式 | **印刷 → PDF**（ブラウザの「PDF に保存」）。追加ライブラリ・サーバー生成なし |
| 保存 | **都度生成のみ**（DB なし）。フォーム状態はクライアントのみ |
| 構成 | `page.tsx`（サーバ）＋ `contract-generator.tsx`（クライアント） |

## 設計判断（なぜこの方式か）

- **印刷→PDF を採用**：撮影業の内部ツールで使用頻度は低く、`@react-pdf` 等のサーバー生成は日本語フォント同梱（数MB）と実装増になる。印刷経由なら依存ゼロ・日本語が完璧・Cloud Run に優しい。1クリック DL が必要になれば後から差し替え可能。
- **DB 非保存（MVP）**：契約は都度入力で足りるため Prisma モデルは追加しない。一覧・再利用が必要になれば `Contract` モデルを足す。

## 使い方

1. 管理画面 → **Contracts** を開く。
2. **案件種別**（撮影＋Web／撮影のみ／Web制作のみ）を選択。**契約した側の条項だけ**を出力する（撮影のみなら撮影枚数・撮影クレジット・出張交通費・生成AI不使用など、Webのみならドメイン・サーバー・解約時の取扱いなど、それぞれ該当分のみ）。納期・納品方法もサービス別に分けて入力。
3. 左フォームに発注者名・委託料・業務内容・振込先などを入力（初期値は「候補者サイト＋撮影」想定でプリセット）。右側プレビューに即反映。
4. **運用・ホスティング条項**（Web案件時のみ）、**政治・選挙特約** はチェックで ON/OFF。条番号は自動で振り直される。
5. 「**印刷 / PDF 保存**」→ 印刷ダイアログで「**PDF に保存**」を選択してダウンロード。

## 案件種別による出し分け（A案：1ジェネレータ＋セレクタ）

撮影用・Web用で別テンプレを二重管理せず、**1つのジェネレータで該当条項のみ出力**する。`projectType`（"photo" | "web" | "both"）から `hasPhoto` / `hasWeb` を導出し、各条項・フォーム項目・プレビュー冒頭文・キャンセル文言・政治特約のWeb専用行をフラグで切替。共通条項（秘密保持・反社・損害賠償・合意管轄等）は1か所で共有。

- **撮影専用**：写真の納品枚数・撮影クレジット・出張交通費・生成AI不使用・写真の軽微調整。
- **Web専用**：ドメイン・サーバー・運用（運用ON時）、および **「サイトの権利帰属・契約終了時の取扱い」**（ドメイン・提供素材・アカウントは甲帰属／継続運用は別途保守運用契約。※制作物の著作権は著作権条による＝留保時と矛盾させない）。
  - **解約時の引渡範囲は著作権の留保/譲渡で出し分け**（匙加減を契約で明示）：
    - **留保（既定）**：稼働可能な状態（公開済みファイル一式）・コンテンツ・サイト保有データ・ドメイン・甲名義設定を渡す。**ソースコード／デザインデータは留保のため対象外**（希望時は別途協議＝譲渡・追加費用）。＝再利用テンプレ等の自分のIPを守りつつ、相手は動くサイトとデータを確保できる。
    - **譲渡**：ソースコード（リポジトリ）・デザインデータを含む一式を渡す。
  - **個人情報**は終了時に甲へ引き渡し、**乙の保有する複製は消去**（個人情報保護）。
- 納期・納品方法はサービス別フラグメントを `／`・`、` で結合し、契約した分だけ印字。

## 乙（個人事業主）保護条項（20原則の反映）

「乙を無限定リスクから守る」観点で条項を追加・強化（条番号はスプレッド除外で動的に詰まるため**相互参照は条名で**記述。本文に `[要確認]` や条番号は印字せず、コード冒頭コメントに法務TODOを集約）。

- **成果物の保証**（新設）：明示仕様への適合のみ保証、無謬性・特定目的適合・第三者環境動作は無保証（as-is）。Web はセキュリティを「努力義務（結果不保証）」と書き分け（個人情報条の措置義務と整合）。
- **契約不適合責任**（新設）：通知期間を検収後 `defectMonths` ヶ月（既定6）に短縮。後出し不具合請求を遮断。
- **損害賠償・免責**（強化）：賠償上限＝支払済み委託料。逸失利益・選挙結果・評判毀損・データ侵害等の間接/特別/結果損害を除外。免責事由は不可抗力条へ集約し参照。
- **不可抗力**（新設）：天災・疾病・パンデミック・広域クラウド障害等は債務不履行を構成しない。`forceMajeureDays` 日継続で解除可。
- **支払遅延時の措置**（新設）：催告＋是正期間を前置して利用許諾停止・公開停止（権利であり義務でない）。**選挙運動期間中の停止は慎重運用**と明記。`delayRate`%（既定14.6）の遅延損害金、残金は検収後60日以内支払。
- **クラウド従量費**（Web）：クラウド利用料・選挙期の一時増強等は委託料に含めず実費パススルー／別途見積（`meteredScope`）。
- **乙からの解約・事業継続**（新設）：乙の退出権（運用部分・予告）、継続不能時は引渡パッケージ提供で解放（後継業者移行は違約でない）。
- **再委託・履行補助者**（新設）：外注・履行補助を許容（選定監督責任は乙）。個人データ扱いの再委託は甲の事前同意。
- **秘密保持**（強化）：一般的知識・技能・汎用フレームワーク/デザインシステムの利用は秘密の対象外（テンプレ再利用の生命線）。
- **個人情報**（日本法準拠に再構成）：甲＝委託元（個人情報取扱事業者）／乙＝委託先。適法取得・同意は甲、監督義務は甲。乙は措置義務のみ。
- **著作権の譲渡カーブアウト**：譲渡時も共通フレームワーク・デザインシステム・ノウハウは譲渡対象外＝乙所有・甲へ非独占ライセンス。
- **第三者の素材・ライセンス**（新設）：OSS/フォント/外部API/ストック写真の継続費・条件順守は甲負担。乙は包括補償しない。
- **肖像権・写り込み**（撮影）：候補者本人以外の同意取得は甲責任。
- **引渡モデルの明記**：動的アプリの実態に合わせ「公開済みファイル一式」表現を廃し、A=専用GCPプロジェクトの請求付替＋IAMオーナー移管／B=git＋pg_dump＋アカウント一覧＋手順書、の2モデルを as-is で一回引渡。引渡後は保管・復元義務なし。
- **反社条項**：表明保証＋無催告解除を多項化。**準拠法＝日本法・管轄＝乙所在地**を明示。
- **政治特約**：乙の責任は技術実装に限定し公選法適法性は甲。候補者の出馬辞退・落選等で中途終了でも既履行分は全額請求でき違約でない。

## 印刷時の挙動

- 管理ナビ（`admin/layout.tsx` の上部バー）と入力フォームは `print:hidden` で非表示。
- プレビュー（`.contract-paper`）のみ印刷。グリッドは `print:block` で1カラム化し、`@page { margin: 16mm }` を適用。

## 著作権条項について（文化庁マニュアル準拠）

第6条（著作権・利用範囲）は、文化庁「誰でもできる著作権契約マニュアル」（写真家向けアレンジ）を踏まえて設計：

- **利用許諾（留保）か譲渡か**を選択。留保時は**利用範囲を明示**（複製・公衆送信・展示・印刷物等のうち定めた範囲に限る）し、**独占／非独占**を指定（規定なき場合は非独占が原則）。
- **譲渡時は二次的著作物（著作権法第27条・第28条）を含む旨を明記**（明記しないと譲渡対象外と推定されるため）。
- **著作者人格権は譲渡不可**。譲渡選択時も「行使しない」とする。トリミング・色調等は軽微な調整のみ可、大幅な改変・合成は協議（同一性保持権への配慮）。
- **クレジット表記（氏名表示権）**の要否を選択。
- 追加条項：契約の解除（催告解除）／権利義務の譲渡制限（利用権の無断譲渡禁止・第63条3項）／契約内容の変更は書面。

## 契約プロセスで使う関連書類（この契約書以外）

| 書類 | 用途・備考 |
|------|-----------|
| 見積書 | 金額の正本。弥生／Misoca で発行（契約書本文は枠組み、金額は見積書に集約）|
| 発注書／注文請書 | 簡易な合意でも可。承諾書形式（一方署名）でも契約は成立 |
| 仕様・業務範囲確認書 | 撮影内容・サイト仕様の詳細を別紙化（スコープ膨張防止）|
| 写真利用許諾書 | 写真の単発利用だけのときの軽い許諾書形式 |
| 納品書・検収書 | 納品・検収の記録 |
| 請求書・領収書 | Misoca で発行 |
| **モデルリリース（肖像権同意）** | **人物撮影で重要。著作権とは別に被写体の肖像権・パブリシティ権の同意が必要な場合がある（マニュアルは著作権のみ）** |
| 撮影許可・会場使用許可 | イベント・施設での撮影時 |
| NDA（秘密保持契約） | 必要に応じて単独締結 |
| **保守運用契約書** | **公開後の運用代行は別契約。`/admin/maintenance` で発行（[17章](./17-admin-maintenance-contract.md)）** |

参考：文化庁「誰でもできる著作権契約マニュアル」／文化庁「著作権契約書作成支援システム」。
